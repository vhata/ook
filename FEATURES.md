- ✓ Books-repo webhook → `/api/webhooks/books/reindex`. Push to `vhata/books` triggers GitHub to POST a signed payload (HMAC-SHA256 over `X-Hub-Signature-256` using `OOK_BOOKS_WEBHOOK_SECRET`); the route verifies the signature and calls the same `reindex()` the admin button uses. Keeps the MCP store fresh without manual nudges; replaces the post-Upstash-provisioning footgun where the store sat empty until someone clicked Reindex. Lives outside the session-cookie proxy matcher — HMAC is the access control.
- ✓ Patch handler frontmatter fidelity. `applyPatch` (the MCP write surface) now performs surgical line-level rewrites on the YAML frontmatter rather than re-serialising the entire block. Adding a single tag produces a commit that touches only that one line, matching the diff preview the user confirmed. `parseMarkdownFile` extracts the raw block via regex (gray-matter v4 doesn't expose `parsed.matter`, so the preserve-raw path was previously a no-op). Six new fidelity tests pin the behaviour.
- ✓ Multi-series schema support. The vault already encodes books with multiple series memberships via `; `-delimited strings (`series: "Discworld, #32; Tiffany Aching #2"`); previously both the renderer and the backfill scripts butchered them. `parseSeriesMemberships(raw)` is now the canonical parser — used by `getAllSeries` (a book lands under each of its series sub-trees) and mirrored into `backfill-see-also.mjs`, `backfill-tags-from-peers.mjs`, and `backfill-see-also-from-tags.mjs` so series-mate detection and diversity dedup count every membership. `parseSeriesField` is preserved as a thin wrapper returning the first membership for back-compat.
- ✓ Per-book page integration tests. The `/books/[slug]` server component now has end-to-end coverage pinning the tier-0/tier-1/tier-2 reveal contract: catalog facts (title, author, status, rating) render in initial HTML; review and notes are gated behind reveal buttons; the share row exposes QR + postcard endpoints; an unknown slug throws `NEXT_NOT_FOUND`. Mocks `next/link`, `next/navigation`, and the data layer; renders the awaited server component output through `@testing-library/react`.
- ✓ Multi-series rendering on per-book pages. Books with `; `-delimited series strings now render as a dot-delimited list of memberships ("Discworld #32 · Tiffany Aching #2") instead of the raw YAML string. Powered by `parseSeriesMemberships`; renders nothing for books without a series.
- ✓ Sub-series detection on `/series` — when every member of series X is also in larger series Y, X gets a "sub-series of Y" badge with an in-page anchor link to the parent. Powered relationally from corpus data; no schema needed. Tiffany Aching, Witches, Watch, etc. now read as Discworld sub-series.
- ✓ Series memberships in the per-book Toc sidebar. The sticky sidebar now leads with a Series section listing every membership and its index, each linking to its `/series` anchor. Stays visible while scrolling deep notes.
- ✓ Gap placeholders on `/series`. Conservative integer-gap detection between known indexes — if the vault has Discworld #1, #3, and #5, the renderer interleaves dashed-border `(not in the vault)` placeholders at #2 and #4. Decimals are ignored (a `#1.5` doesn't fill the surrounding integers); never invented past the lowest or highest known index.
- ✓ Hybrid Hardcover series rosters. New `scripts/backfill-series-rosters.mjs` (run via `make vault-series-rosters` / `make vault-series-rosters-apply`) fetches each series the vault knows about from Hardcover's GraphQL API and writes `_meta/series-rosters.json` to the vault. The `/series` renderer reads that cache at request time: when present, missing entries render with their canonical title + author from Hardcover; the header denominator becomes the canonical total ("3 of 41 read" instead of "3 of 3 read in vault"). When no roster exists for a series, falls back to the corpus-only gap detection. Internet only at user-initiated `make` time; build stays offline-clean.
- ✓ `/robots.txt` and `/sitemap.xml` via Next 16 file conventions (`src/app/robots.ts`, `src/app/sitemap.ts`). Robots allows the public surfaces, disallows `/vault-health`, `/schema`, `/api/`. Sitemap enumerates every public route — home, log, stats, series, shelf, discover, tags, triage, changelog, random — plus every book page, every bingo year (`/print/[year]`), and every stats year. Per-book `lastModified` reads from the existing `lastEdited` field.
- ✓ Canonical Make targets for vault hygiene — `make vault-lint`, `make vault-backfill` (dry-run), `make vault-backfill-apply` (writes). The umbrella runs every backfill in dependency order: `source`, `tags` (Open Library), `tags-from-peers` (corpus-derived), `see_also` (series + author), `see_also-from-tags` (tag-Jaccard).
- ✓ Two corpus-internal backfill scripts that derive enrichment from the vault alone — no network. `scripts/backfill-tags-from-peers.mjs` proposes tags for thinly-tagged books by tallying their series-mates, same-author books, and see_also peers under signal-source-aware thresholds (series-fraction ≥ 0.66, author-fraction ≥ 0.8 with min count 3, see_also count ≥ 3). `scripts/backfill-see-also-from-tags.mjs` extends `see_also` via tag-Jaccard similarity (min 3 shared tags, Jaccard ≥ 0.4) with diversity caps (≤ 1 entry per series, ≤ 2 per author) so a heavily-tagged series can't crowd out other recommendations. Both default to dry-run.
- ✓ `/admin/audit` — passkey-gated audit log of recent vault commits. Shells out to `git log` against the cloned `.vault/` (same `execFile` shape as `getLastEditedMap`), parses the last 50 commits with TAB-delimited fields plus `--shortstat` for files-changed counts, and renders a list with date, subject, author, file count, short SHA, and a deep-link to the commit on `vhata/books`. Unauthenticated requests bounce to `/admin`. Cross-linked from the admin console as a quiet "Recent activity →" affordance next to Reindex / Sign out. Closes the "did that commit go through?" loop without leaving the site for GitHub. Empty-state when `.vault/` isn't a git repo (dev without `BOOKS_DEPLOY_KEY`). Parser pinned by unit tests. Footer carries the store's `lastReindex` stamp (relative time) so a stale-looking commit list can be disambiguated from a stale reindex at a glance; silent-omit when the key is absent (dev without a store, fresh deploy before the first reindex).
- ✓ Webhook-driven vault hygiene. After the books-repo webhook reindexes the MCP store, it fires a `repository_dispatch` event on the ook repo to kick off `.github/workflows/vault-hygiene.yml`, which checks out both repos, runs `make vault-lint` (fail-loud), runs the offline-fast backfills with `--apply` (`source`, `tags-from-peers`, `see_also`, `see_also-from-tags`, `see_also-bidirectional` — Open Library tags stay operator-initiated), and pushes a single `Auto-hygiene:` commit if anything changed. Webhook handler filters non-main pushes and short-circuits the dispatch when the head commit message starts with `Auto-hygiene:` so the workflow's own commits don't trigger redundant runs. Concurrency-grouped so rapid pushes coalesce to one run.
- ✓ Navigable `/series` — sticky left-rail TOC on desktop, horizontal scroll-strip of series chips on mobile, and `<details>`-based collapse-by-default for series with more than four rendered rows (members + roster-missing placeholders). Sub-series indent under their parent in the TOC. `?expand=all` and `?collapse=all` query params drive an Expand-all / Collapse-all toggle near the page header — server-rendered, no JS island. Real `id` anchors so back/forward and copy-link sharing work; `scroll-mt-6` keeps the header visible when jumping into a section.
- ✓ Interactive prompt-to-apply on every backfill script. When stdin is a TTY and the dry-run produced pending changes, each script prints the summary and asks `Apply these N changes? [y/N]`; on yes, it writes from the changeset already computed (no re-deriving, no re-fetching from Open Library or Hardcover). Non-TTY stdin (CI, pipes, redirected input) skips the prompt and exits dry-run cleanly. Shared helper `scripts/lib/maybe-prompt-apply.mjs` keeps the prompt logic out of every script; covers `backfill-source`, `backfill-tags`, `backfill-tags-from-peers`, `backfill-see-also`, `backfill-see-also-from-tags`, `backfill-see-also-bidirectional`, `backfill-series-rosters`, and `backfill-hardcover-books`.
- ✓ `SeriesHashOpener` client island on `/series` — opens any `<details>` whose `id` matches `window.location.hash` (and re-runs on `hashchange`). Closes the loop where a per-book Toc link to `/series#series-discworld` would otherwise scroll to a closed disclosure. Pure side-effect, renders nothing.
- ✓ Kindle `My Clippings.txt` one-shot importer (`scripts/import-kindle-clippings.mjs`, run via `make vault-import-kindle FILE=…` / `make vault-import-kindle-apply FILE=…`). Parses the clippings file (UTF-8 with BOM and UTF-16-LE both supported, CRLF tolerated), fuzzy-matches each highlight's Kindle title to a vault directory (high precision: exact normalised equality, then conservative substring containment with length floors), and appends new entries into the matched book's `quotes.md` under a `## From Kindle` block (Notes go to `## Notes from Kindle`; Bookmarks are dropped). Each entry carries a stable 16-char hash in an HTML comment so re-running the importer never duplicates. Unmatched titles land in `_meta/kindle-unmatched.md` for triage. Pure parsing, matching, dedupe, and rendering helpers live in `scripts/lib/kindle-clippings.mjs` and are pinned by unit tests.
- ✓ Pages-derived stats on `/stats/[year]` — Topline now carries a "Longest" metric (the longest finished book of the year by Hardcover-`pages`, with title + author hint), and a 12-bar pages-per-month chart sits below the weekend-split section. Both surfaces degrade silently when the Hardcover cache has no `pages` for the books in question (no metric, empty chart skipped). Source data is the existing `_meta/hardcover-books.json`; no new external fetches.
- ✓ "Pages" total in the `/stats/[year]` Topline strip — sum of Hardcover `pages` across every finished book in the year that has a paged record, rendered with comma separators next to Avg rating, with a "from N of M books with page data" hint so the coverage gap is visible. Tile is omitted entirely when no finished book in the year has a paged Hardcover record (silent degrade, same shape as the Longest metric).
- ✓ Reading-velocity ETA on Currently-Reading cards — when the home-page reading book has `hardcover.pages` and the user has finished books with `pages` data in the last 3 months (falling back to 12 months when 3 is empty), the card adds an "≈ N days at your pace" caption. Pace is `total pages / number of distinct days that had a finish`, not days elapsed in the window — counts the days the reader actually closed a book. Skipped under the time-machine lens (pace doesn't make sense in past-views) and absent for books without paged Hardcover records.
- ✓ `scripts/backfill-hardcover-ids.mjs` — copies `hardcoverSlug` and `hardcoverId` from the existing `_meta/hardcover-books.json` cache into each book's frontmatter as `hardcover_slug` / `hardcover_id`. Pure cache-to-frontmatter (no network), surgical line-level writes (no whole-block re-serialisation), per-field skip so a manually-set slug or id is never overwritten even when the cache disagrees. Wired up as `make vault-hardcover-ids` (dry-run) / `make vault-hardcover-ids-apply`. Closes the gap where the renderer's outbound external-link row was silently skipping Hardcover for every book the user hadn't manually annotated.
- ✓ `/now` — minimal embeddable "what I'm reading right now" surface. Three signals only: currently-reading book(s), the most recently finished book, and the current ongoing reading streak (rendered when ≥ 2 days). No nav, no controls bar, no footer chrome — `SiteChrome` (a small `usePathname`-aware wrapper inside the root layout) suppresses the global wrapper for this route so the page can be iframed cleanly into a personal homepage or shared as a now.html-style link. `revalidate = 300` (5-minute stale-while-revalidate); `robots: noindex,nofollow`; no `?at=` time-machine lens (live signal only). Streak helper `getCurrentReadingStreak` lives next to `getCurrentlyReading` in `src/lib/books.ts`; counts back from today (or yesterday if today has no events yet) over started/finished/manual-log dates.
- ✓ Vault → Hardcover one-way status sync. New `scripts/sync-hardcover-status.mjs` (run via `make vault-hardcover-sync` / `make vault-hardcover-sync-apply`) pushes the vault's `status`, `rating`, `started`, and `finished` fields up to the operator's Hardcover account via GraphQL `insert_user_book` / `update_user_book` mutations and `upsert_user_book_reads` for read-session dates. Goodreads' write API has been dead since 2020-12-08; Hardcover is the only viable target. Idempotent through two layers: a local `_meta/hardcover-sync-state.json` cache short-circuits no-op re-runs, and a remote diff check before every mutation prevents double-writes when the cache is stale. Vault is the source of truth — a flip from finished back to tbr still pushes — but Hardcover ratings finer than the vault's integer scale are preserved when the difference is ≤ 0.5. Wired into `.github/workflows/vault-hygiene.yml` so every vault push automatically syncs when the `HARDCOVER_TOKEN` repo secret is populated.
- ✓ "What others said" disclosure on per-book pages — top 2-3 short, high-rating, non-spoiler reviews from Hardcover, surfaced inside a click-to-reveal section that sits alongside the existing review/quotes/synopsis tier-1 reveals. New `scripts/backfill-hardcover-reviews.mjs` (run via `make vault-hardcover-reviews` / `make vault-hardcover-reviews-apply`) queries Hardcover's `user_books` table — Hardcover has no `book_reviews` root type, reviews live on the per-user shelf row — filters server-side for `has_review`, `rating ≥ 3`, `review_length` 80..600, and `review_has_spoilers: false`, and writes the top 3 by `likes_count` to `_meta/hardcover-reviews.json` keyed by vault slug. Bodies arrive as HTML and are stripped to plain text before caching. Per-book opt-out via `hide_external_reviews: true` in frontmatter; books without qualifying reviews don't render the section at all. Same operator-initiated, vault-committed-cache discipline as the other Hardcover scripts.
- ✓ MCP commits carry a `via ook-admin/<session-id>` trailer so `/admin/audit` can distinguish writes that flowed through the in-process MCP tools (the passkey-gated `/admin` console + the external MCP HTTP transport) from direct pushes to `vhata/books`. The trailer is appended once at module load — the session id is a 7-char slice of `crypto.randomUUID()`, stable for the life of the function instance — and rides at the end of the user-supplied commit message, separated by a blank line. Idempotent: re-tagging an already-tagged message is a no-op. The audit parser reads the body via a NUL-record `git log -z` with `%b` in the format and extracts the session id when the trailer is on the final non-empty line; the audit page renders a small "via MCP" chip on those rows. Chip says "MCP" rather than "/admin" because both surfaces flow through the same tools. Parser pinned by unit tests covering subject-only mentions, mid-body mentions, and idempotent re-tag.
- ✓ Finish-flow pullquote + rating gate on `/admin`. When the user reports finishing a book, the admin agent asks for a pullquote and a star rating before staging the status flip, then bundles all three changes into a single `propose_patch` call so one commit lands on `vhata/books`. The gate lives in the agent's system prompt (natural language — the tool layer doesn't enforce it), is default-on but operator-overrideable ("just mark it finished, skip the rating"), and treats explicit refusal of both questions the same as a tab close: the status flip is lost, by user-stated design. To make the ask/answer/ask/answer/commit shape actually work, the `/api/admin/agent` endpoint now round-trips an opaque `state` handle (the Anthropic SDK `messages` array) and the `AdminConsole` carries it forward across follow-up submits — single-turn for fresh requests, multi-turn for clarifications. Patch-handler bundling is pinned by a `applyPatch` unit test; the system-prompt instruction block is string-pinned in `admin-agent-prompt.test.ts` (behavioural coverage of the gate itself would need a mock Anthropic transport — manual-test recipe documented inline in the same file).
- ✓ `/admin/backfill` — passkey-gated "fill the gaps" view that picks 3-5 small, skippable questions per visit about older finished books missing one piece of metadata each. Three kinds at launch: a rating prompt for finished books with no rating; a one-or-two-sentence review prompt for finished books rated ≥ 4 with no `review.md`; a yes/no would-reread prompt for 5-star finishes where the field is null. Candidates from each kind are shuffled independently then interleaved round-robin so a single visit always sees a mix; dedupe by slug keeps any one book from appearing twice on a visit. Saves post a structured `CommitPatchInput` straight to the existing `/api/admin/agent/commit` route — no Claude tokens burned for a "set rating=4" deterministic update — so every backfill commit picks up the same `via ook-admin/<id>` trailer and audit-log entry as a `/admin` console write. Skip is pure client-side state; a "that's all for this visit" footer surfaces when every card is resolved. Cross-linked from the admin console next to Recent activity. Pure derivation from `getAllBooks` via `src/lib/admin/backfill.ts`; no network at request time. `pickQuestions` is exposed for tests and pinned end-to-end on candidate selection, kind interleave, slug-level dedupe, and the empty-corpus fallback.
- ✓ Colored unified-diff output in vault-script dry-runs. Each frontmatter-mutating backfill (`source`, `tags`, `tags-from-peers`, `see_also` series+author / from-tags / bidirectional, `hardcover-ids`) now prints a per-book block shaped as a unified diff — `→ <slug>` header, then a red `- <old line>` / green `+ <new line>` pair for line replacements (or just the green `+` for pure insertions). Plain `-` / `+` markers carry the same signal when stdout isn't a TTY (CI logs, redirected output, pagers without `-R`), so golden files and grep stay stable. Honours the de-facto-standard `FORCE_COLOR` / `NO_COLOR` env vars so child processes can opt in without a real terminal. Shared helper at `scripts/lib/diff-format.mjs`, pinned by unit tests + an integration test on `backfill-hardcover-ids` that spawns the script with `FORCE_COLOR=1` and asserts the green-wrapped insertion lines flow through.
