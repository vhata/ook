# Features

Feature ledger. Grouped by milestone. One line per entry. Plain language.

Legend: ✓ shipped · ⋯ in progress

## Milestone — initial work

- ✓ Home page renders Currently Reading, Recently Finished, and the 2026 Bingo grid from the local vault path.
- ✓ Bingo grid shows title + first author per square, with a gold star on done squares and a `N / 24 squares done` counter.
- ✓ Per-book pages at `/books/[slug]` rendering body markdown plus optional review and quotes. Linked from home-page book cards and bingo cells with a known book. Every book is reachable.
- ✓ **Tiered spoiler model.** Tier 0 (catalog: title, author, status, rating, bingo, dates) always visible. Tier 1 (synopsis, review, quotes) server-rendered, click-to-reveal, per-session persistent. Tier 2 (deep notes) fetched client-side from `/api/books/[slug]/notes` only after explicit opt-in — never in initial HTML, so search engines don't index spoilers.
- ✓ TBR / Re-Read Aspirations section on the home page renders `_meta/tbr.md` body.
- ✓ Bare YAML dates in book frontmatter (`finished: 2026-02-20`) parsed into `YYYY-MM-DD` strings, fixing recently-finished sort order.
- ✓ Per-book pages show which bingo squares the book claims, with a link back to the bingo grid.
- ✓ Per-book pages render the book's tags as small badges.
- ✓ Per-book document title is `<Title> — <Author> · ook`; layout title template "%s · ook".
- ✓ Custom 404 page consistent with site styling (root + `/books/[slug]` segment variants).
- ✓ Global footer with link back to the GitHub repo.
- ✓ Open Graph metadata for the site title, description, and type.
- ✓ Stats line under the home-page H1 (`N reading · M recently finished · X / Y bingo`).
- ✓ Visual overhaul to Claude Design's D3 prototype: paper-and-ink + charcoal palette, Source Serif 4 + Inter Tight + IBM Plex Mono, rust accent, bordered stats strip, large hero "Reading now" card, finished grid with covers, bingo with id chips and per-square cover slots, TBR with sub-pile filter pills, sticky-TOC book pages with pullquote and cross-links.
- ✓ Theme toggle (light / dark / auto) persisted in localStorage via `useSyncExternalStore`.
- ✓ Frontmatter extensions: `cover`, `pullquote`, `see_also`, derived `lastEdited` from git log.
- ✓ TBR pile parser (`## Wanted`, `## Re-Read Aspirations`, etc.) into typed entries; falls back to raw markdown body when piles are empty.
- ✓ `/log` route: reading log grouped by month, derived from each book's `started`/`finished` dates.
- ✓ Spoiler markdown directive (`:::spoiler ... :::` via remark-directive) renders click-to-reveal blur on per-book pages.
- ✓ Mobile responsive: bingo becomes a horizontal scroll-strip (5×5 stays sacred); finished cards stack horizontally; stats compress to 2-col with short labels; controls bar icon-only.
- ✓ Open Library cover URLs in frontmatter for all bingo squares + per-book directories. Bingo cell tooltip shows "title — author" on hover.
- ✓ Vercel deploy live at https://b-ook.vercel.app. Vault split into private `vhata/books` repo with deploy key + webhook on push for auto-rebuild.
- ✓ `bin/book` Node CLI in the vault for deterministic frontmatter mutations + `book list` (with vault / bingo / Goodreads sources, filters by status, tag, bingo, rated) + `book covers` (HTML grid of Open Library editions in the browser) + `book cover` (set chosen URL) + `book import-goodreads` (parse CSV, write to `_meta/goodreads.md`).
- ✓ Auto-promote bingo and Goodreads-only entries to vault directories on first mutation. Verification prompt for GR-claimed status / rating / date.
- ✓ Multi-year bingo support: current year derived from the highest `_meta/bingo-YYYY.md` filename; per-book pages attribute to whichever card actually references the book.
- ✓ External-link row on per-book pages — "View on Goodreads · Hardcover · Storygraph · Bookwyrm" rendered from optional `goodreads_id`, `hardcover_slug`, `storygraph_slug`, `bookwyrm_url` frontmatter fields. Missing fields produce no link; nothing is guessed.
- ✓ `/stats/[year]` route: yearly reading stats derived from frontmatter — finished count, abandoned, started-in-year, average rating with histogram, top tags, top authors, would-reread count. `/stats` index redirects to the most recent year with activity. Header includes a year-picker across all years that have data. Reachable from the controls bar.
- ✓ Manual log entries from optional `_meta/log.md` — non-book events (notes, "committed to bingo", "added to TBR", reread, progress) merged with frontmatter-derived started/finished events on `/log`. Schema: `## YYYY-MM-DD` headings, bullets prefixed with bold kind labels (`**Note** — body`).
- ✓ Per-route Open Graph share images via Next 16 `opengraph-image.tsx` file convention. Home card shows the wordmark + currently-reading + last-finished slots; per-book card shows cover, title, author, status, rating, finish date. Source Serif 4 fetched from Google Fonts at generation time so share-card typography matches the live site.
- ✓ `/random` route — redirects to a random finished book; reachable via the 🎲 button in the controls bar. Empty-vault fallback redirects home.
- ✓ `/feed.xml` Atom and `/feed.json` JSON Feed of finished books — discoverable from `<head>` so RSS readers auto-detect. Pullquote (or first review paragraph) used as each entry summary; finish date as `<updated>`/`date_published`. Cached for an hour.
- ✓ `/series` browser — every book with a `series` frontmatter, grouped by name, ordered by `#N` index when present. Each row shows cover thumb, status, rating, finish date. Reachable from the controls bar.
- ✓ Reading-day heatmap on `/stats/[year]` — GitHub-contribution-style calendar grid, 7×52 cells, intensity ramped by event count per day (started + finished + manual log entries). Tooltip shows date and count.
- ✓ "On this day" strip on the home page — surfaces reading-log events from past years that share today's month-and-day (started/finished/notes). Renders only when there's something to show; year tag + kind label + linked book title.
- ✓ `/stats` index now renders "Years in reading" — each year's first finished book and last finished book with rating + finish date, linking through to the year's full stats. Falls through to `/stats/[currentYear]` only when no year has data yet.
- ✓ `/discover` route — top-N most-connected book pairs in the vault, scored by see-also links, shared series, shared author, and overlapping tags. Per-pair reason chips explain the connection. Pure vault derivation, no external recommender.
- ✓ `/tags` index + `/tags/[tag]` — tag taxonomy with sized tag cloud, per-tag co-occurrence chips, and a per-tag book grid. Tag chips on per-book pages now link into the matching tag page.
- ✓ Reading-drought banner on `/log` — when the most-recent event is more than 21 days old, the page heads with a soft "X days since the last event — quiet stretch" callout.
- ✓ Polish: breathing accent dot on the home-page masthead (4-second opacity loop, gentler than the reading-now pulse), and a redesigned 404 page across both root and per-book routes — site palette, dropped-book line-art SVG, restored typography.
- ✓ "Threads" sidebar on per-book pages — top 3 algorithmically-similar books from `getSimilarBooks`, alongside the existing user-curated "See also" links. Each thread row shows the strongest reason (see-also · series · author · tag) and links into the full `/discover` index.
- ✓ Rotating pullquote on the home page — random `pullquote` from any finished book that has one, surfaced as a small italic block between the stats strip and the rest of the page. Re-rolled per page-load.
- ✓ Inline "X days quiet" markers between adjacent log entries on `/log` — companions to the top-of-page drought banner, surfacing in-month and cross-month gaps that exceed the 21-day threshold.
- ✓ Bingo `done`-ness derived from the bound book's status. Single source of truth — the YAML `done:` field is no longer trusted; a square is done iff its bound book is finished.
- ✓ ESLint encoding of "vault is read-only from src/" — `no-restricted-imports` + `no-restricted-syntax` ban `fs.writeFile`, `fs.appendFile`, `fs.rm*` (and `node:fs` / `fs/promises` equivalents) anywhere under `src/**`. The prebuild SSH-key writer in `scripts/` is out of scope.
- ✓ React-component test scaffold — `@testing-library/react` + `happy-dom` under vitest with `// @vitest-environment happy-dom` per-file. Initial coverage on `Spoiler` and `RevealSection` (the load-bearing tier-1/tier-2 reveal primitives).
- ✓ Serendipity card on the home page — a quiet "Remember this?" sidebar surfacing one finished book whose finish date is more than a year old. Re-rolled per page-load like the rotating pullquote. Cover thumb on wider screens; year-count callout in the kicker.
- ✓ Weekday vs weekend split on `/stats/[year]` — under the heatmap, two-row bar chart comparing per-day reading-event rate on weekdays vs weekends, with a "weekends run 2.3× hotter" / "even split, day for day" callout for the skew direction.
- ✓ Rating-over-time chart on `/stats` — inline-SVG dot plot of every rated finished book against finish date, with a rolling-average accent line (window scales with corpus size) and year ticks on the x-axis. Reveals taste drift, harshening, kindening. Renders only when there are at least four rated finishes.
- ✓ "Stuck" badge on per-book pages and home finished cards — surfaces books that earned the reader's full attention. Rule: finished + has a review + has quotes + (rating ≥ 4 OR would-reread). Three signals because any one alone is too easy to clear.
- ✓ "The wall" cover mosaic on `/stats/[year]` — every cover of every finished book in the year, tiled into a single block at the bottom of the year stats page. Auto-fill grid sized to the available width (≥64px tiles), in finish-date order. Hover-scales each tile; tooltips name the book.
- ✓ Per-book QR endpoint at `/books/[slug]/qr` — 512×512 PNG that encodes the canonical book URL in the paper-and-ink palette. Slug is validated against the vault; unknown slugs 404 so the endpoint can't mint QRs for arbitrary URLs. Cached aggressively (1d browser, 7d edge).
- ✓ Foxing-by-age CSS filter on finished-book covers — subtle sepia + slight contrast loss scaled by years-since-finish. Applied to home `FinishedCard` thumbs and the `/stats/[year]` cover mosaic. Capped at sepia(0.32) and contrast(0.92) so even ancient books stay legible. Computed in the parent server component so React's purity rule on `Date.now()` doesn't fire.
- ✓ Seasonal accent palette drift — the rust accent shifts gently across the year (rust in winter, ochre in spring, slate-blue in summer, forest in autumn). Implemented via a `<style>` tag in the root layout that overrides `--accent` and `--accent-soft` based on the server's current month. Paper-and-ink stays constant.
- ✓ Marginalia rendering for the quotes section — when `quotes.md` is revealed, blockquotes render with hand-drawn-feeling pencil-bracket SVGs on either side instead of the standard left-border. Blockquotes elsewhere on per-book pages are unaffected.
- ✓ Postal-stamp flourish on per-book pages — finished books get a small SVG stamp in the cover corner: title up top, star rating in the middle, author + postmark date at the bottom, perforated border. Slightly rotated. Doesn't replace the cover; sits as a thumbnail-sized seal next to it.
- ✓ Word-frequency cloud on `/stats` ("What you keep saying") — top words across every `review.md`, filtered against a stop-word list, sized by frequency. Renders only when the corpus has at least six non-singleton words.
- ✓ "Days in" indicator on Currently Reading cards — derived from each book's `started` date. Reads as a small uppercase caption under the author line.
- ✓ Finishing-patterns callout on `/stats` — consecutive-finish ordered pairs that occur ≥2 times across the reading log. Surfaces habits like "always reads A before B."
- ✓ Time-machine view at `/?at=YYYY-MM-DD` — renders the home page as it would have looked on a past date by filtering the current vault frontmatter through a date lens. Currently-reading, recently-finished, and bingo all re-derive against the lens date; the secondary surfaces (TBR, on-this-day, rotating quote, serendipity) hide so the page reads cleanly. Top-of-page banner with "back to today" link.
- ✓ `/shelf` — bookspine view of every finished book. Vertical SVG spines side-by-side, coloured from a hash of the first tag (or first author when untagged), title running upward, author at the foot, decorative bands at top + bottom. Sortable by finish date / author / rating / title.
- ✓ `/print/[year]` — printable year bibliography. Clean centre-anchored typographic layout — index number, title, author, finish date, optional pullquote, would-reread loop mark. `@media print` rules strip nav, force serif, and set A4 margins so Cmd-P → PDF produces a real-paper artefact.
- ✓ `/changelog` — what's been edited in the vault recently. Books grouped into Monday-anchored ISO weeks by their `lastEdited` date (already derived from `git log` on the reference file). Each row shows date, title link, status. Pure derivation from existing data — no new tooling needed in the vault.
- ✓ `/triage` — pool of books to decide on. Renders manual recommendations from `_meta/triage.md` (same shape as `tbr.md` — H2 piles + bulleted entries) plus imported-but-not-yet-fleshed-out Goodreads entries (any title in `_meta/goodreads.md` that doesn't yet have a vault directory). Read-only; the user does the actual triaging in the vault.
- ✓ `scripts/promote-goodreads.mjs` — bulk-mints per-book vault directories from `_meta/goodreads.md` entries that haven't been promoted yet. Shelf → status mapping (read=finished, currently-reading=reading, to-read=tbr), series extracted from "(Name, #N)" parenthetical, ratings/dates carried through. Defaults to dry-run with a shape summary; `--apply` writes.
- ✓ `scripts/import-triage.mjs` — generic CSV → `_meta/triage.md` converter. Loose column mapping (title required; author/why/source/pile optional). Supports `--append` to merge with an existing triage file.
- ✓ `/vault-health` page + `scripts/vault-lint.mjs` — same checks two ways. Surfaces missing-but-expected frontmatter (no `finished` date on a finished book, broken `see_also` references, no rating where one's conventionally expected). Severity tiered as error / warning / info; the route renders a faceted view, the script does CLI output for pre-push audits.
- ✓ Procedural covers for books without a `cover:` URL. Replaces the old crosshatched placeholder with a hash-coloured SVG: gradient background from the same eight-colour palette as `/shelf` (selected by hash of the title), oversized initials in the centre (skipping articles like "the"), title in small caps at the foot. Reads as intentional rather than missing-data.
- ✓ Per-book postcard PNG at `/books/[slug]/postcard.png` — 1200×800 sharable artefact via `next/og`. Cover floated left, title + author + pullquote on the right, accent stripe at the bottom, ook mark in the top-right corner, URL + date in the footer. Cached aggressively (1d browser, 7d edge).
- ✓ Share row on per-book pages — small "↓ QR · ↓ postcard" affordances under the external-link row, surfacing the QR and postcard endpoints that previously required URL-guessing. Each opens in a new tab.
- ✓ Corpus-level vault-health checks — an orphan detector (books with no incoming `see_also` references and no bingo binding) and an asymmetric-see_also detector (`A → B` where `B` doesn't link back). Both surface as info-level findings on `/vault-health` and through `vault-lint --severity info`. Same checks two surfaces, mirrored between `src/lib/vault-health.ts` and `scripts/vault-lint.mjs`.
- ✓ `scripts/backfill-see-also-bidirectional.mjs` — closes the loop on `see_also` reciprocity. For every `A → B` in the vault where `B` doesn't link back, proposes adding `A` to `B`'s `see_also` (additive only — never bumps existing entries; respects MAX_SEE_ALSO). Designed to run after the other see_also backfills; pure derivation, idempotent, dry-run by default. Wired into the `vault-backfill` umbrella.
- ✓ Component test coverage for `DeepNotes` and `Cover` — the tier-2 fetch-on-click reveal contract is now pinned end-to-end (mocked fetch, error retry, sessionStorage rehydrate), and the procedural-cover fallback is pinned at the hash-colour and initials-derivation level. `vitest.config.ts` added to mirror the `@/*` tsconfig alias so components can be tested without rewriting their imports.
- ✓ `/schema` route — auto-documented corpus shape. For every field the renderer reads, shows coverage (X of Y books populated, with a horizontal bar), kind, and up to three sample values. Companion to `/vault-health`: that page shows what's missing per book; this one shows the schema as a whole. Marked `noindex`. Powered by `src/lib/schema.ts` (pure derivation from typed Book records).
- ✓ Canonical Make targets for vault hygiene — `make vault-lint`, `make vault-backfill` (dry-run), `make vault-backfill-apply` (writes). The umbrella runs every backfill in dependency order: `source`, `tags` (Open Library), `tags-from-peers` (corpus-derived), `see_also` (series + author), `see_also-from-tags` (tag-Jaccard).
- ✓ Two corpus-internal backfill scripts that derive enrichment from the vault alone — no network. `scripts/backfill-tags-from-peers.mjs` proposes tags for thinly-tagged books by tallying their series-mates, same-author books, and see_also peers under signal-source-aware thresholds (series-fraction ≥ 0.66, author-fraction ≥ 0.8 with min count 3, see_also count ≥ 3). `scripts/backfill-see-also-from-tags.mjs` extends `see_also` via tag-Jaccard similarity (min 3 shared tags, Jaccard ≥ 0.4) with diversity caps (≤ 1 entry per series, ≤ 2 per author) so a heavily-tagged series can't crowd out other recommendations. Both default to dry-run.
